# Reference match

> Make the build match the mock on every axis you name, not just "looks close."

## Goal (the what)

- **Objective:** Make a built screen match its design mock.
- **Success criteria:** The build matches the mock on spacing, type scale, color, and layout.
- **Constraints:** Match the design exactly, do not invent improvements mid-pass.
- **Context:** The mock, the build, and the four axes that count.

## Loop (the how)

- **Think:** Compare the build to the mock on one axis at a time.
- **Act:** List every difference, then fix them.
- **Observe:** Re-compare and see which differences remain per axis.
- **Reflect / Plan:** Carry matched axes forward, keep working the rest.
- **Stop when:** Every axis (spacing, type scale, color, layout) shows zero differences.

## Starter prompt

```
Compare this built screen to the design mock across four axes: spacing, type scale,
color, and layout. List every difference under each axis. Fix them, then compare
again. Keep looping until each axis has zero differences. Show the remaining diff list
after every pass. Stop only when the build matches the mock on all four axes.
```
