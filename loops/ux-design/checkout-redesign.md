# Checkout redesign

> Cut a bloated flow down to the essential steps, with the reasoning documented.

## Goal (the what)

- **Objective:** Redesign the mobile checkout flow.
- **Success criteria:** Steps reduced from 6 to 3, high-fidelity Figma screens delivered, and the reasoning for each change documented.
- **Constraints:** Keep every legally or operationally required field. Do not remove guest checkout.
- **Context:** The current 6 step flow, the design system, and known drop-off points.

## Loop (the how)

- **Think:** Look at the current steps, decide what can be merged, deferred, or cut.
- **Act:** Draft the 3 step flow, build the high-fidelity screens, write the rationale per change.
- **Observe:** Check that no required field was dropped and that the count is actually 3.
- **Reflect / Plan:** Restore anything required, rework steps that still feel heavy.
- **Stop when:** The flow is 3 steps, all required fields remain, screens are high-fidelity, and each change has a documented reason.

## Starter prompt

```
Redesign the mobile checkout flow. Current flow is 6 steps: [list]. Reduce it to 3
without dropping any required field or removing guest checkout. Produce high-fidelity
screens and document the reasoning for each change. Check the result: confirm it is 3
steps and all required fields remain. Stop only when it is 3 steps, every required
field is present, and each change is justified.
```
